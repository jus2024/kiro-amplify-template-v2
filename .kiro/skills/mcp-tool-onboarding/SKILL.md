## 目的

- 不要な MCP の複雑さを避ける
- 明確な価値がある場合のみ MCP を追加する
- 認証、利用方法、検証を明確に文書化する

## ワークフロー

1. MCP が本当に必要か確認する
2. 対象の機能を説明する
3. 認証とシークレット管理の要件を特定する
4. MCP ツールの使用方法を説明する
5. 最初の統合は最小限に保つ
6. 検証とフォローアップの強化手順を文書化する

## 出力の期待事項

- 目的の記載
- セキュリティ上の考慮事項
- ローカルテストのアプローチ
- フォローアップのクリーンアップや強化のアイデア
